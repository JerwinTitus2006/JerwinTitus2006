# Contributing to Jerwin's Profile Repository

Thank you for your interest in contributing! This repository manages my professional developer profile and automated workflows.

## How to Contribute

### 1. Reporting Bugs & Requesting Features
- Search existing issues to ensure it hasn't been reported.
- Use the appropriate **Issue Template** to create a bug report or feature request.
- Provide detailed steps, expected vs. actual outcomes, and screenshots if applicable.

### 2. Submitting Pull Requests (PRs)
- Fork the repository and create your branch from `main`.
- Ensure your changes follow the premium, minimal design language.
- Run a local validation of any workflow files (`.github/workflows/*.yml`) if you modify them.
- Submit a PR referencing the issue it solves, using the **Pull Request Template**.

## Style Guide
- **Minimalist & Clean**: Keep designs modern and spacing elegant.
- **Workflow Integrity**: Workflows must compile, run, and use official actions.
- **No Emojis**: Avoid excessive emojis to maintain a professional, Apple-level aesthetic.
